  Recent high-profile supply chain attacks, from SolarWinds to the xz-utils backdoor, have exposed a fundamental flaw in modern software security: the gap between build-time security and runtime integrity. We have made significant strides in securing the software supply chain with frameworks like SLSA and tools like Sigstore. These are essential, but they only verify the integrity of software \textit{before} it is deployed.

  This whitepaper introduces Attestium, an open-source framework we built to bridge this gap. Attestium provides continuous, verifiable runtime integrity for server-side applications by leveraging Trusted Platform Modules (TPMs) and a unique layered architecture. It offers a practical, scalable solution for achieving end-to-end trust, ensuring the code you run is the code you trust.
